## Top 7 finalistas v3 — universales

Criterios: convergencia entre agentes (señal real) + factibilidad técnica para 3 devs/60h + ataca patrón ganador histórico + apoyo de tendencia macro 2026.

### 🏆 #1 — Portfolio Rebalancer / Yield Router

**Convergencia 8/8 (TODOS)** | Patrón ganador C (Cleopetra $15K, Plutus $15K, Voltr $7.5K) | Complejidad Media

Variantes propuestas convergen en: agente que monitorea APYs/posiciones cross-protocol, decide rebalance dentro de policy declarada, ejecuta con guardrails. Variantes específicas: Self-Hedging Vault (delta-neutral via lending+perps), Cross-Chain Arb Router (bridges + swaps cuando spread > fees), LP Autopilot (par concreto).

- **Stack:** RPC streaming + 2-3 protocolos DeFi + LLM para reasoning + smart contract policy
- **Por qué gana:** convergencia máxima + patrón ganador histórico + tendencia macro 2026 (DeFAI maduro)
- **Riesgo:** competencia alta (Cleopetra ya ganó). Mitigar con scope acotado y feature diferenciador (ej. delta-neutral, multi-protocol, intent-driven)

### 🥇 #2 — Multi-Agent A2A Negotiation Marketplace

**Convergencia 6/8** | Validación: Anthropic experiment 186 deals + patrón D ganador | Complejidad Media-Alta

Marketplace donde un agente "comprador" y un agente "vendedor" negocian precio/términos vía A2A protocol, llegan a acuerdo, liquidan vía escrow on-chain. Logs auditables del proceso de negociación.

- **Stack:** Agent framework × 2 + escrow Anchor program + protocolo de mensajería + simple oracle
- **Por qué gana:** validación externa fuerte (Anthropic), demo entretenida, multi-agente premia jurados
- **Riesgo:** la "negociación" puede ser frágil. Mitigar con protocolo de resolución determinístico

### 🥇 #3 — Reputation/Identity Passport (KYA)

**Convergencia 5/8** | Patrón ganador B (Latinum $25K, AgentiPy $15K) + tendencia a16z 2026 | Complejidad Media

Pasaporte on-chain con historial verificable de un agente AI: track record (PnL, slashing, errores documentados), capabilities declaradas, spending limits, owner. Cualquier protocolo puede consultar antes de aceptar tx del agente o delegarle fondos.

- **Stack:** cNFTs (compresión) + agent framework + verificadores + UI consulta + simple SDK
- **Por qué gana:** "pico y pala" puro, todos los demás equipos lo necesitan, tendencia a16z 2026 explícita
- **Riesgo:** "wow demo" menor. Mitigar con dashboard que muestre simulación de delegación de fondos

### 🥈 #4 — Compliance Watchdog / Circuit Breaker

**Convergencia 5/8** | Patrón ganador F (JailbreakMe $20K) | Complejidad Media

Agente monitorea wallets/treasury en tiempo real, detecta interacciones anómalas (aprobaciones a contratos sospechosos, drenado, patrones rug), pausa vía social recovery o multisig veto. Variante: pre-tx screening contra sanctions/risk score.

- **Stack:** RPC streaming + LLM para análisis + smart contract pause/multisig + UI alertas
- **Por qué gana:** tendencia macro fuerte ($45M pérdidas, 35% no puede apagar agents), narrativa concreta (Step Finance hack)
- **Riesgo:** menor agencia "wow" que un trader. Mitigar mostrando ataque en vivo + kill-switch parándolo

### 🥈 #5 — Conversational Wallet / Intent-to-Execution

**Convergencia 5/8** | Patrón ganador A (The Hive $60K) | Complejidad Media

Chat NL sobre wallet: "consigue mejor yield para mis stables sin riesgo de despeg", agente parsea intent, simula multi-protocolo, ejecuta tx atómica con confirmación opcional y slippage cap. Variantes: stake/swap/transfer multi-step, onboarding a protocolo.

- **Stack:** LLM con tool-calling + 4-5 protocolos DeFi + wallet connect + simulador
- **Por qué gana:** patrón ganador A trajo $60K (premio mayor histórico), reduce fricción real, killer use case según Coindesk
- **Riesgo:** competencia (Hive ya ganó). Mitigar con dimensión adicional: voz, mobile, vertical específico

### 🥉 #6 — Multi-Agent Debate (Bull/Bear/Risk Supervisor)

**Convergencia 5/8** | Validación científica peer-reviewed | Complejidad Media-alta

Vault DeFi donde 3 agentes (alcista, bajista, supervisor de riesgo) debaten en lenguaje natural antes de cada rebalanceo. Debate completo guardado on-chain como auditoría. Voto por consenso.

- **Stack:** 3 LLMs personas distintas + agent framework + protocolo voto on-chain + yield aggregator
- **Por qué gana:** validación científica peer-reviewed (BlackRock+Columbia 2026), demo escénico fuerte, verifiable reasoning natural
- **Riesgo:** lógica de debate frágil. Mitigar con time-box + supervisor con poder de veto

### 🥉 #7 — DAO Proposal Triage Bot

**Convergencia 3/8** | Asistivo (humano-en-loop) | Complejidad Baja

Lee propuestas DAO, resume, modela impacto on-chain simulando, detecta colusión votantes, publica scorecard pre-voto. NO vota, solo asiste.

- **Stack:** RPC indexer + LLM + simulator + UI dashboard
- **Por qué gana:** complejidad **más baja** = más realista para 60h, patrón "asistivo con humano-en-loop" que literatura 2026 dice que funciona
- **Riesgo:** menor "wow" → menor premio top, pero alta probabilidad de demo limpia

### Lección metodológica final

- **v1** introdujo sesgo por TRACKS/SPONSORS en prompts → "convergencia 8/8 voice+Li.Fi" era pura amplificación del input
- **v2** introdujo sesgo por DEMOGRAFÍA en prompts ("Universidad Javeriana, Colombia, estudiantes, primer hackathon") → emergieron CampusDAO/CongresoMirror/AcademiaProof/GuardiánBeca como "convergencia" — pero NO sobreviven en v3
- **v3** con prompts mínimos revela las convergencias REALMENTE robustas: yield management, A2A marketplaces, reputation/KYA, compliance, conversational wallets, multi-agent debate

**Para futuros research agents en este proyecto:** los prompts deben ser estrictamente sobre el dominio del problema. Información sobre el equipo (demografía, ubicación, idioma, experiencia previa) se aplica POST-síntesis cuando se rankean ideas, NO en el prompt de descubrimiento.

## Aplicación post-síntesis: unfair advantages reales del equipo

Los unfair advantages del equipo (no inyectados en los prompts) son útiles para **escoger entre las ideas universales**, no para **inducir cuáles ideas existen**.

**Unfair advantages identificados:**
- Idioma español nativo + inglés
- Contexto LATAM (regulación, mercados, casos de uso locales)
- Posiblemente acceso a comunidad universitaria (testers/usuarios)

**Aplicación a las top 7 v3:**

| Idea | Unfair advantage aplicable | Cómo |
|------|---------------------------|------|
| #1 Portfolio Rebalancer | Bajo | Mercado universal, español es nice-to-have pero no diferenciador |
| #2 A2A Negotiation Marketplace | Bajo | Mercado universal B2B/agente-agente, no language-specific |
| #3 Reputation Passport (KYA) | Medio | Si hay onboarding/UI español, ayuda adopción LATAM. Pero la infra es universal |
| #4 Compliance Watchdog | Medio-alto | Compliance es REGIÓN-específico (regulación). LATAM/Colombia tiene casos uso únicos (sanciones, cumplimiento bancario, AML local). **Posible diferenciador real** |
| #5 Conversational Wallet | **Alto** | "Wallet conversacional en español" es UX gap real. Adopción cripto LATAM crece, herramientas siguen siendo gringas. **Diferenciador fuerte con bajo costo técnico extra** |
| #6 Multi-Agent Debate | Bajo | Mercado universal de DeFi |
| #7 DAO Proposal Triage | Medio | Algunas DAOs son ESpañol-first o LATAM-specific (si las hay). Más nicho |

**Aplicación práctica:**
- Si pick es **#1 Portfolio Rebalancer**: build universal, el equipo no aporta unfair advantage particular
- Si pick es **#3 Reputation Passport**: build universal con UI bilingüe como nice-to-have
- Si pick es **#4 Compliance Watchdog**: explorar AML/sanctions LATAM como ángulo (p.ej., reglas SFC Colombia, listas SDN para LATAM) — potencialmente alto diferenciador
- Si pick es **#5 Conversational Wallet**: **ATAR el español/LATAM como feature core** ("primer wallet conversacional español-nativo para Solana") — alto leverage
- Si pick es **#7 DAO Proposal Triage**: investigar si hay DAOs LATAM específicas

**Recomendación final post-todo:**

Pick principal: **#1 Portfolio Rebalancer / Yield Router** — la convergencia 8/8 sin sesgo es señal incomparable, ataca patrón ganador histórico ($15K varios), tendencia macro fuerte, scope acotable a 60h.

Pick alternativo si quieren maximizar unfair advantage español/LATAM: **#5 Conversational Wallet en español** — alto leverage de skills del equipo, también patrón ganador histórico (The Hive $60K), demo memorable. Mayor riesgo de competencia técnica pero el ángulo regional está sub-atendido.

Antes de comprometerse:
1. Validar contra Colosseum Copilot (https://colosseum.com/copilot)
2. Mappear a tracks oficiales del Dev3pack desde `docs/hackathon-info.md` y `docs/member-resources.md` (NO al revés — escoger idea por mérito y luego ver qué track encaja, no escoger por track)
3. Si la idea no encaja en ningún track Dev3pack pero el equipo la cree mejor, evaluar trade-off
